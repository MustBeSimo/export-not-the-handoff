The agent layer

Generate → export code → manually hand off
Prompt → source change → versioned, reviewable output
illustrative sketch · text in, rendered output in place
When the artifact is code, an agent can edit the source and run the pipeline. The shift isn't autonomous deploy; it's that creation, revision, and rendering work on the same source.

In file-based creative workflows, sharing your work usually meant freezing it. You exported a file so it could travel, and it went rigid the moment it left the tool. Web runtimes and source-driven deployment changed that: some work can now travel as a live system instead of a frozen deliverable. Export isn't gone, broadcast, film, regulated docs, and approved records still need an immutable version. So the new problem isn't how to export the work. It's deciding which outputs stay live, and which must be locked as an approved record.
