# The Export Button Is No Longer the Handoff

A live, single-file interactive artifact supporting the LinkedIn post *"The Export Button Is No Longer the Handoff"*, the sequel to *HTML Is the New After Effects*. It demonstrates the HTML-source / browser-runtime example directly; the Slidev, Rive, HyperFrames, and agent points elsewhere in the post are illustrated, not run here.

**The point:** this is one self-contained `index.html` file (vanilla JS, no libraries, no build step). It generates animated motion graphics live in your browser and lets you edit them in real time. The source is this HTML file; the page you see is the live artifact, not a media file someone exported and sent you.

## The thesis, in one line

In file-based creative workflows, sharing your work usually meant freezing it into a file that went rigid the moment it left the tool. What changed isn't that export disappeared; it's that web runtimes and source-driven deployment made live delivery practical. Export is not gone (broadcast, film, regulated documents, and offline work still need a file of record), but where runtime is the destination, some work can travel as a live system instead. The new question becomes a governance one: which outputs stay live, and which must be frozen as an approved record.
